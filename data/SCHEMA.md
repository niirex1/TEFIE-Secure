# Data schema

Replace every file in this folder with your **real** experimental outputs in the
formats below, then run `python reproduce.py` from the artifact root. The files
shipped here are SYNTHETIC smoke-test data (generated by
`src/make_synthetic_data.py`) and do **not** reproduce the manuscript's values.

All classification metrics, ROC curves, and AUCs are computed from per-instance
predictions, so the artifact never stores a metric you cannot recompute from
these inputs.

---

## `predictions_classification.csv`
Per-instance held-out predictions for every task and model. Drives the accuracy
table and the ROC figures.

| column        | type   | meaning                                            |
|---------------|--------|----------------------------------------------------|
| `task`        | str    | `reentrancy` \| `timestamp` \| `loops`             |
| `model`       | str    | model name (e.g. `TEFIE-Secure`, `VDM-AEI`, ...)   |
| `seed`        | int    | training seed (0..4)                               |
| `instance_id` | int    | row id within the (task, model, seed) group        |
| `y_true`      | 0/1    | ground-truth label (1 = vulnerable)                |
| `y_score`     | float  | model score in [0,1] for the positive class        |

## `predictions_ablation.csv`
Same schema as above with `model` replaced by `variant`
(`full`, `w/o cross-addr. typing`, `w/o feature selection`,
`w/o contextual embeddings`, `w/o preprocessing`, `w/o residual refiner`).
Drives the ablation table.

## `runtime.csv`
Direct timing/resource measurements (not derived from predictions).

| column               | type  | meaning                              |
|----------------------|-------|--------------------------------------|
| `model`              | str   | model / tool name                    |
| `mode`               | str   | `source` or `bytecode`               |
| `latency_ms_p50`     | float | median per-function latency (ms)     |
| `latency_ms_p99`     | float | p99 per-function latency (ms)        |
| `throughput_inst_s`  | float | instances per second                 |
| `peak_vram_gb`       | float | peak GPU memory (GB)                 |
| `params_m`           | float | parameter count (millions)           |

## `static_points.csv`
Single (FPR, TPR) operating point per static tool per task, plotted as markers
on the ROC figures.

| column | type  | meaning                          |
|--------|-------|----------------------------------|
| `task` | str   | task name                        |
| `tool` | str   | `Slither` \| `Mythril`           |
| `fpr`  | float | false-positive rate at its point |
| `tpr`  | float | true-positive rate at its point  |

## `shap.csv`
Mean absolute SHAP attribution per feature group per task; drives
`feature-importance.pdf`.

| column          | type  | meaning                                                          |
|-----------------|-------|------------------------------------------------------------------|
| `task`          | str   | `Reentrancy` \| `Timestamp` \| `Loops`                           |
| `feature_group` | str   | `call-edge`, `state-update`, `loop-structure`, `block-variable`, `normalization` |
| `mean_abs_shap` | float | mean absolute SHAP value for that group                          |

## `realworld.csv`
RQ5 post-2023 mainnet contracts.

| column        | type   | meaning                                                 |
|---------------|--------|---------------------------------------------------------|
| `contract_id` | str    | on-chain address                                        |
| `vuln_type`   | str    | class label / family                                    |
| `y_true`      | 0/1    | 1 if a disclosed exploit exists for this contract       |
| `y_score`     | float  | model score in [0,1]                                    |
| `disclosed`   | 0/1    | 1 if publicly disclosed (defines the positive set)      |
