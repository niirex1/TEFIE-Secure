# TEFIE-Secure — reproduction artifact

This artifact reproduces the tables and figures of the TEFIE-Secure manuscript
from raw experimental outputs. It contains the full reproduction pipeline; the
result *values* come entirely from the contents of `data/`.

## Important: shipped data is synthetic

The files in `data/` are **synthetic smoke-test data** generated by
`src/make_synthetic_data.py`. They let the pipeline run end-to-end but **do not
reproduce the values reported in the paper**, by design — the scores are drawn
from generic class-separated distributions, not calibrated to any reported
number. To reproduce the manuscript, replace `data/` with your real experimental
outputs following `data/SCHEMA.md`, then re-run.

## Layout

```
reproduce.py              one-command entry point
requirements.txt
src/
  metrics.py              F1/precision/recall/FPR/FNR + ROC/AUC from raw scores
  tables.py               accuracy, ablation, runtime, real-world tables
  figures.py              ROC (x3) and feature-importance figures
  workbook.py             results.xlsx (metrics as in-cell formulas)
  make_synthetic_data.py  synthetic smoke-test generator (NOT paper values)
data/                     inputs (synthetic now; schema in data/SCHEMA.md)
datasets/                 ESC/VSC access: provenance, fetch script, synthetic examples
paper_figures/            the figures currently in the manuscript, for reference
outputs/                  generated tables (.csv), figures (.pdf), results.xlsx
```

## Datasets (ESC / VSC)

The paper evaluates on the public ESC and VSC corpora of Zhuang et al. (IJCAI
2020). The artifact does **not** redistribute them — the source repositories
carry no explicit licence — so `datasets/fetch_dataset.py` downloads the
authentic files directly from the authors' repository, pinned to an immutable
commit and checksum-verified:

```bash
python datasets/fetch_dataset.py     # reentrancy (ESC) + loops (VSC)
python datasets/load_dataset.py      # print verifiable dataset statistics
```

`datasets/synthetic_contracts/` provides three clearly-labelled synthetic
contracts (one per task) for an offline demo. Full provenance, citation, the
license note, and the full-corpus source are in `datasets/DATASETS.md`.

## Run

```bash
pip install -r requirements.txt
python reproduce.py --synth     # generate smoke-test data, then build everything
# or, once data/ holds your real outputs:
python reproduce.py
```

Outputs are written to `outputs/`:
- `table_accuracy.csv` → **Table II** (detection accuracy across classes)
- `table_runtime.csv` → **Table III** (runtime and resource profile)
- `table_ablation.csv` → **Table IV** (per-stage ablation)
- `table_realworld_bytype.csv`, `table_realworld_summary.csv` → **Table V** (post-2023 mainnet)
- `reentrancy-ROC.pdf`, `timestamp-ROC.pdf`, `loops-ROC.pdf` → **Figure 2** (ROC)
- `feature-importance.pdf` → SHAP feature-importance figure
- `results.xlsx` — one workbook with all table and figure data

(Table I in the manuscript is a qualitative enforcement-point summary and is not
data-derived, so it is not regenerated here.)

This artifact is a revision of an earlier one whose harness scraped the paper PDF
and replotted hand-made curve spreadsheets instead of computing results; see
`REVISION_NOTES.md` for exactly what changed and why.

## How the numbers are produced

- **Accuracy / ablation / real-world**: computed from per-instance predictions
  (`y_true`, `y_score`) by thresholding and counting TP/FP/FN/TN, then applying
  the standard metric formulas. In `results.xlsx` these metrics are written as
  **Excel formulas over the counts**, so every reported value is auditable
  in-cell and recomputes if the counts change.
- **ROC / AUC**: a score-threshold sweep over the same predictions.
- **Runtime, memory, parameters, SHAP, static-tool points**: direct measurements
  read from `data/`; they are inputs, not derived quantities.

Nothing in `src/` hardcodes a value reported in the paper.

## Reproducing the paper

1. Replace each file in `data/` with your real outputs (`data/SCHEMA.md`).
2. `python reproduce.py`
3. Compare `outputs/` against the manuscript tables and `paper_figures/`.
